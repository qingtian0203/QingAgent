## 【幻灯片 3】先说：本地大模型是怎么跑起来的？

核心概念：量化（Quantization）

- 大模型本质是一个巨大的数学权重文件，动辄几十 GB
- 量化 = 把高精度数字压缩成低精度，文件变小，速度变快，精度小幅损失
- 常见精度：4bit（最小最快）、8bit（质量更好）、16bit（原版）
- Gemma4 26B 4bit 只需要约 12G 内存，普通 M 系列 Mac 装得下

运行框架：Ollama

- Ollama 是一个本地大模型管家：一键下载、一键启动、一键对话
- 就像 Docker 对代码，Ollama 对模型：统一打包、统一运行
- 启动后，它在本机开一个 HTTP 端口，任何程序都可以往这个端口发请求、拿到 AI 的回答

## 【幻灯片 6】第一步：看到——AI 怎么「看」屏幕？

截图 → 转成图片数据 → 发给视觉大模型 → 模型返回坐标

**发给大模型的请求长这样：**

```json
{
  "model": "gemma-4-26b-a4b-it-4bit",
  "messages": [{
    "role": "user",
    "content": [
      {
        "type": "text",
        "text": "这是一张微信截图，请找到【晴天的名字】的中心位置，返回 {rx:0-1000, ry:0-1000}"
      },
      {
        "type": "image_url",
        "image_url": {"url": "data:image/png;base64,iVBORw0KGgo..."}
      }
    ]
  }],
  "stream": false,
  "max_tokens": 512
}
```

**为什么返回 0-1000 而不是像素？**

- 屏幕分辨率不同，像素坐标每台机器不一样
- 千分比是归一化的，1000 用整数，比 0.153 更稳定，不会被模型输出成科学计数法

模型返回：`{"rx": 153, "ry": 420}`
含义：目标在图片宽度 15.3% 处，高度 42% 处

## 【幻灯片 7】坐标怎么转成屏幕上真实的点击位置？

```
截图区域 rect = (100, 50, 800, 1000)
   意思：窗口左上角在屏幕 (100,50)，宽 800px，高 1000px

模型返回：rx=153, ry=420

屏幕 x = 100 + (153/1000) × 800 = 222
屏幕 y =  50 + (420/1000) × 1000 = 470

→ pyautogui.click(222, 470)
```

就是这么简单的一个数学换算，没有魔法。

## 【幻灯片 8】撞墙：点击经常点歪——然后一条小红书救了我

**问题：** 模型看到的是整张大图，按钮可能只占 0.01% 的像素，找不准很正常

**正发愁的时候，刷到一条小红书：**
> Gemma4 在 oMLX 上能提速 30%，M 系列 Mac 满血发挥

- MLX 是苹果自己开源的深度学习框架，专为 Apple Silicon 设计
- 充分利用统一内存架构，CPU/GPU/Neural Engine 协同，不需要数据搬运
- oMLX 把 MLX 包成和 Ollama 完全一样的 HTTP 接口

**实测结果：**

| 框架 | 单次图片识别 |
|------|------|
| Ollama | ~10 秒 |
| oMLX | ~0.7 秒 |

切换成本：**代码里只改一个 URL 地址**，其他完全不动。

## 【幻灯片 9】用速度优势换精度——三重放大定位

速度从 10 秒降到 0.7 秒，我开始想：**能不能用这个速度优势加强识别精准度？**

试了两个方案：

- 方案 A：同一张图识别三次，取平均值 → 仍然不准，三次的偏差加在一起
- 方案 B：三重放大（像用 8 倍镜瞄准）→ **精准无比**

**三重放大流程：**

```
第一段：全图发给模型 → 粗定位（大概在右上角）

第二段：裁剪目标附近 300×300 小图再发
       → 模型视野缩小，目标占比更大，纠偏更准

第三段：裁剪 80×80 指甲盖大小再发
       → 模型几乎只看到目标本身，精确锁心

三段总耗时：< 3 秒，误差 ≤ 3 像素
```

每段模型看到的图像尺寸相同，但目标占的比例越来越大——**识别越来越准**。

## 【幻灯片 10】第二步：理解——AI 怎么知道你要做什么？

有了眼睛，接下来是大脑。这步反而最简单。

用一个纯文字大模型（Planner）理解你说的话：

**你输入：** 「给晴天小米发条微信让他记得打卡」

**Planner 返回：**
```json
{
  "steps": [{
    "step": 1,
    "app": "微信",
    "intent": "send_message",
    "slots": {
      "contact_name": "晴天小米",
      "message": "记得打卡哦"
    },
    "description": "发微信给晴天小米"
  }]
}
```

大模型在这里扮演的角色是**翻译官**：把人话翻译成机器能执行的 JSON。
它不关心后面怎么执行，只负责理解意图、输出格式正确的结构化指令。

## 【幻灯片 11】第三步：去做——osascript 是 macOS 自带的遥控器

**两套工具组合：**

**① pyautogui（Python 库）**
- 控制鼠标移动、点击、键盘按键
- 把视觉模型返回的坐标直接变成真实点击

**② AppleScript + osascript（macOS 原生）**
- 苹果在 1993 年就内置的脚本语言，专门让程序之间互相控制
- macOS 大多数 App 都暴露了「遥控接口」

**一个技巧：进程级快捷键注入**

有时 App 的焦点跑到内嵌网页里（比如微信的公众号文章），
普通的快捷键就打到了错误的地方。

解决方案：直接把快捷键注入进程队列，完全绕过焦点：

```applescript
tell application "System Events"
    tell process "WeChat"
        keystroke "f" using command down  -- 不管焦点在哪，直达微信
    end tell
end tell
```

## 【幻灯片 15】当前系统的真实架构

```
你（浏览器输入指令）
    ↓ HTTP POST
Python HTTP 服务（localhost:8888）
    ↓
Planner（文字 LLM）
  → 解析意图，输出 JSON 步骤数组
    ↓
Skill 路由（字典查找）
    ↓ 按 app 名找对应 Skill
应用层 Skill（WeChat / Browser / System / Antigravity...）
    ↓ 继承 BaseSkill
基础层 Skill（BaseSkill）
  ├── activate()       激活应用窗口
  ├── screenshot()     截图
  ├── find_and_click() 三段式定位 → 点击
  └── read_content()   截图 → 视觉读取文字
    ↓
actions 模块
  ├── pyautogui → 鼠标 / 键盘
  └── osascript → 系统级操作
    ↓
执行日志实时推送 → 浏览器显示
截图结果返回 → 用户确认（HITL 安全机制）
```

每个 Skill 就是一个独立的「操作员工」，微信 Skill 只管微信，浏览器 Skill 只管浏览器。

## 【幻灯片 18】如果你也想上手，核心步骤和代码

**第一步：装 oMLX，跑本地模型**

```bash
pip install mlx-lm
# 下载模型（约 12G）
mlx_lm.convert --hf-path google/gemma-3-27b-it \
                --mlx-path ./gemma4-4bit -q
# 启动服务，兼容 OpenAI 格式，监听 8000 端口
mlx_lm.server --model ./gemma4-4bit --port 8000
```

**第二步：Python 发第一个请求给本地模型**

```python
import requests

response = requests.post("http://localhost:8000/v1/chat/completions", json={
    "model": "gemma4-4bit",
    "messages": [{"role": "user", "content": "你好，用一句话介绍你自己"}],
    "stream": False,
    "max_tokens": 200
})

print(response.json()["choices"][0]["message"]["content"])
# 就这 10 行，本地 AI 开始为你回答问题
```

**第三步：带图片识别坐标**

```python
import base64

with open("screenshot.png", "rb") as f:
    img_b64 = base64.b64encode(f.read()).decode()

response = requests.post("http://localhost:8000/v1/chat/completions", json={
    "model": "gemma4-4bit",
    "messages": [{
        "role": "user",
        "content": [
            {"type": "text", "text": "找到【发送按钮】的位置，返回JSON: {rx:0-1000, ry:0-1000}"},
            {"type": "image_url", "image_url": {"url": f"data:image/png;base64,{img_b64}"}}
        ]
    }],
    "max_tokens": 50
})

coords = response.json()["choices"][0]["message"]["content"]
# 拿到坐标，换算，点击
```

**第四步：用 osascript 控制系统（激活微信示例）**

```python
import subprocess

subprocess.run(["osascript", "-e", '''
    tell application "WeChat" to activate
    tell application "System Events"
        tell process "WeChat"
            keystroke "f" using command down  -- 打开搜索
        end tell
    end tell
'''])
# 微信置于前台，搜索框打开，剩下的用 pyautogui 输入联系人名
```

从第二步到第四步，**一个最小可用的 AI Agent 就完成了**，几十行代码。

## 【幻灯片 19】结语

不需要懂深度学习，不需要 GPU 服务器，不需要 API Key。
**只需要一台 M 系列 Mac 和几十行 Python。**
